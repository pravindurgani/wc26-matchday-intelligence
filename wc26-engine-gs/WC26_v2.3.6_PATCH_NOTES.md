# WC26 Engine v2.3.6 — Patch Notes

**Release type:** six-closure end-to-end fix on top of v2.3.5 — 4 CRIT + 1 HIGH + 1 MED.
**Schema/XLSX changes:** none. Drop-in replacement.
**Compatible with:** `https://wc26-matchday-intelligence.vercel.app/` (R32 kickoff 2026-06-28, T-4d).
**Why it exists:** v2.3.5 polished the *output* of the knockout staking matrix on the Bets sheet but left every *input* unwired — Bets!L:N seeding was missing, the entire Matchday sheet stopped at row 75, the protection allow-list locked the operator out of the new rows, and the onEdit snapshot handler had a hard-coded `mdRow > 75` guard. End-to-end the documented operator UX ("the ONLY tab you use is MATCHDAY") was unreachable for all 32 knockout matches.

---

## CRIT-A — `extendToKnockouts()` never seeded Bets!L:N (operator-odds mirror)

**What was wrong (v2.3.5 and earlier):** xlsx ships shared formulas `ref="L3:L73"`, `M3:M73`, `N3:N73` (the Matchday → Bets mirror for typed 1/X/2 odds). Knockout rows 74-105 fall outside that ref → empty. v2.3.5 P0-A added a 36-cell staking matrix downstream of L/M/N (cols O..AQ/AR..AV/AZ..BA) but never seeded L:N itself. Empty L → `O = IF(L="","",1/L)` = `""` → `R = ""` → `S/T/U = ""` → … → `AH = "· enter odds"` → `AJ = 0`. **Net pre-fix: zero bets sized on every R32 match even though the staking matrix was "complete."**

**What v2.3.6 does:**
- Adds a 4th block `L_N` to `_knockoutStakingFormulas_()` mirroring the xlsx master row 2 byte-exact: `=IF(Matchday!E76="","",Matchday!E76)` / `F` / `G`.
- Adds a 4th seeding block in `extendToKnockouts()` with the same hasAny-preservation idiom used for O..AQ/AR..AV/AZ..BA.
- Status message extended: `staking O..AQ=N AR..AV=N AZ..BA=N L..N=N row(s) seeded`.

**Validation receipt:**
- `crita_L74_anchor_E76`: `=IF(Matchday!E76="","",Matchday!E76)` ✓
- `crita_L105_anchor_E107` (last R32 row) ✓
- Compared against xlsx Bets!L2 byte-exact ✓.

## CRIT-B — `extendToKnockouts()` never extended the Matchday sheet (rows 76-107)

**What was wrong (v2.3.5):** Matchday sheet `sheet5.xml` has cells defined out to row 1000 but only rows 1-75 carry content (group stage). Rows 76-107 are empty `<c>` shells. Even with CRIT-A above, `Matchday!E76` resolved to `""` because no one had typed there — and there was no operator-facing surface for them to type into (no auto-mirrored team names, no decision-message formula).

**What v2.3.6 does:**
- Adds `_matchdayKnockoutRowFormulas_(r)` returning a 21-cell row matching the xlsx Matchday!row 4 shape exactly:
  - 11 auto-mirror formulas: A/B/C/D (match #, date, time, label) + H/I/J/K (pick/decision/stake/msg from Bets) + M (decision-message multi-clause) + S (audit-message mirror) + U (CLV formula using rm-anchored T/P).
  - 10 operator-input nulls: E/F/G (odds H/D/A), L (team-news Y/N flag), N/O/P/Q/R (bet management), T (closing odds), V/W (future).
- Adds a Matchday extension block in `extendToKnockouts()` with the same conservative preservation idiom (if ANY cell already holds a formula OR an operator-typed value, the whole row is preserved untouched). Uses `setValues` (handles both formulas and literal blanks).
- Status message extended: `Matchday rows seeded=N preserved=N`.

**Validation receipt:**
- `critb_row_length_21` ✓ (21-cell row matches xlsx A..U coverage)
- `critb_A_matchno_anchor`: `=Bets!A74` ✓
- `critb_U_clv_matchday_anchor`: `=IF(OR(T76="",P76=""),"",P76/T76-1)` ✓ (uses rm=76, not r=74)
- `critb_M_msg_uses_L76`: team-news clauses reference `L76` (Matchday row), Bets refs use row 74 ✓
- `critb_r105_anchor_A107` (last row): `=Bets!A105` + `T107/P107` ✓
- All 10 operator-input cells return `null` ✓.

## CRIT-C — `applyProtections()` Matchday allow-list locked operator out at row 75

**What was wrong (v2.3.5):** allow-list was `Matchday: ['E4:G75', 'L4:L75', 'N4:R75', 'T4:T75']`. Even with Matchday rows 76-107 now seeded (CRIT-B), the operator couldn't type odds into E76:G107 — the cells were inside the script-protected region.

**What v2.3.6 does:** bumps each Matchday range to row 107: `['E4:G107', 'L4:L107', 'N4:R107', 'T4:T107']`.

**Validation receipt:**
- `critc_matchday_E_extended_to_107` ✓
- `critc_no_residual_row75_in_matchday`: no `E4:G75` literal anywhere ✓.

## CRIT-D — `onEdit()` snapshot handler had `mdRow > 75` hard-coded guard

**What was wrong (v2.3.5):** when operator marks Matchday!N76="Y" to log a placed knockout bet, onEdit's `if (mdRow < 4 || mdRow > 75) continue;` silently no-op'd. The Bets!AW:AY snapshot block (decision/stake/pick frozen at bet-placement time) never captured anything for the 32 knockout matches → CLV tracking and post-hoc audit columns showed nothing.

**What v2.3.6 does:** bumps the upper guard to 107.

**Validation receipt:**
- `critd_onedit_upper_bound_107` ✓
- `critd_no_residual_row75_in_onedit` ✓.

## HIGH — `_knockoutStakingFormulas_` AU formula missing `&` concat operators (carried fix)

**What was wrong:** v2.3.5 line 1563 (the AU "audit message" formula) had `&AP' + r + '" vs engine` and `&AV' + r + '" but engine` — missing the `&` between the cell reference and the next string literal. In Sheets this produces `#ERROR!` whenever the operator marks a knockout bet as placed (AN flips to "Y"). Display-only (zero money impact) but it would show `#ERROR!` on every settled bet in column AU.

**What v2.3.6 does:** added `&` after each cell reference: `&AP' + r + '&" vs engine` and `&AV' + r + '&" but engine`. Now byte-exact against xlsx master AU2.

**Validation receipt:**
- `high_AU_byte_exact_xlsx_master` ✓ (475 bytes, exact match against extracted xlsx AU2 formula)
- `high_AU_AP_concat_operator` + `high_AU_AV_concat_operator` ✓.

## MED — `installEngine()` now auto-runs `extendToKnockouts()`

**What was wrong (all prior versions):** `installEngine()` ran refreshAll → refreshMatchContext → snapshotExisting → installAutoRefresh → applyProtections. It did NOT run extendToKnockouts. Operator had to discover the menu item "WC26 Engine ▸ Extend to knockout stage" and run it manually before R32. Easy to forget at T-4d.

**What v2.3.6 does:** inserts `extendToKnockouts()` between `refreshMatchContext()` and `snapshotExisting()`. Idempotent (re-runs preserve existing rows). Install toast updated: "WC26 Engine v2.3.6 installed (knockouts pre-seeded)."

**Validation receipt:**
- `med_installEngine_auto_extends` ✓ (regex matches `extendToKnockouts()` call inside `installEngine()` body before `installAutoRefresh()`).

2. Paste contents of `WC26_Engine_AppsScript_v2.3.6.gs` (replace existing).
3. Save (⌘S / Ctrl+S).
4. Reload spreadsheet so the `onOpen()` menu re-installs.
5. Menu: **WC26 Engine ▸ First-time setup (run once)**. Approve OAuth scopes if prompted.
6. Confirm install toast: "WC26 Engine v2.3.6 installed (knockouts pre-seeded)."
7. Scroll Matchday to row 76 — should see R32 fixture labels appearing as the Python feed populates team names (post group-stage conclusion 2026-06-27).
8. Type 1/X/2 odds in E76/F76/G76. BET? signal renders in I76 within seconds.
