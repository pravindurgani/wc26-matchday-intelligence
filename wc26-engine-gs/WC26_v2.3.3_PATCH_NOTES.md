## CRIT #1 — `refreshIntel()`: v2.3.2 fan-out branch was dead code

### What was wrong (v2.3.2)

v2.3.2 introduced a fan-out branch for `match_id=null` (team-level,
tournament-wide) intel entries. The gate was:

```js
const mid = Number(a.match_id);
if (isFinite(mid)) {
  // match-level branch
} else {
  // team-level fan-out branch (NEW in v2.3.2)
}
```

But `Number(null) === 0`. So every team-level entry produced `mid=0`,
passed `isFinite`, and was routed into the match-level branch. The
fan-out branch was never reached. `_pushPart_(0, team, ...)` wrote
the breakdown to the ghost match #0, which has no Bets row, so the
delta was silently discarded.

Net effect: identical to the pre-v2.3.2 silent drop. The Belgium
−22.6, Algeria −11.08 etc. injury/stats_proxy deltas still did not
reach the Bets sheet — they just took a different invisible path
to get dropped.

This is also why v2.3.2's Python "self-validation" claimed success:
the Python mirror reproduced the same `Number()` coercion against
its own dict (Python `int(None)` raises, so the Python script
shortcut around the bug bilaterally). The bug was only visible
through the production sheet behaviour.

**Lesson banked in CLAUDE.md prompting convention:** never
self-validate inside the pane that wrote the patch. The AUDIT
pane catches what self-checks miss.

### What v2.3.3 does

`refreshIntel()` now uses `_num_()` instead of `Number()`:

```js
function _num_(v) {
  if (v === null || v === undefined || v === '') return NaN;
  const n = Number(v);
  return isFinite(n) ? n : NaN;
}
```

`_num_(null)` returns `NaN`, which fails `isFinite`, so the team-level
fan-out branch is the only path for null/undefined/empty match_ids.

### Validation

`test_harness.mjs` now ships a synthetic intel-gate regression
(`intel_gate_v233`) that drives `_num_` against eight payload shapes
and asserts the routed branch:

| payload | rawMid | branch | expect |
|---------|--------|--------|--------|
| `47` | 47 | match-level | match-level ✓ |
| `null` | NaN | team-level | team-level ✓ |
| `undefined` | NaN | team-level | team-level ✓ |
| `""` | NaN | team-level | team-level ✓ |
| `"47"` | 47 | match-level | match-level ✓ |
| `"abc"` | NaN | team-level | team-level ✓ |
| `0` | 0 | match-level | match-level ✓ |
| `true` | 1 | match-level | observed (not asserted) |

`all_ok: true`. Had this test existed for v2.3.2, the dead-code
fan-out would have been caught at harness time.

## CRIT #2 — `refreshOutrights()` blew away operator-typed col L

### What was wrong (v2.3.2)

The Outrights refresh did:

```js
out.getRange(dataStart, 1, prevLast - dataStart + 1, 18).clearContent();
out.getRange(dataStart, 1, rows.length, 11).setValues(rows);
```

Col L holds operator-typed bookie odds. `clearContent()` over cols
A:R wiped every L cell, then the rewrite only repopulated A:K. Net:
on every tick (10 min), every typed bookie odd vanished.

### What v2.3.3 does

Snapshot col L by team before clear; restore onto the same-team
row after the (re-sorted) write:

```js
const oddsByTeam = {};
if (prevLast >= dataStart) {
  const prevRange = out.getRange(dataStart, 1, prevLast - dataStart + 1, 12).getValues();
  for (let i = 0; i < prevRange.length; i++) {
    const team = prevRange[i][0];
    const odds = prevRange[i][11];
    if (team && odds !== '' && odds !== null && isFinite(_num_(odds))) {
      oddsByTeam[String(team)] = odds;
    }
  }
  out.getRange(dataStart, 1, prevLast - dataStart + 1, 18).clearContent();
}
out.getRange(dataStart, 1, rows.length, 11).setValues(rows);
for (let i = 0; i < rows.length; i++) {
  const team = rows[i][0];
  if (team && Object.prototype.hasOwnProperty.call(oddsByTeam, team)) {
    out.getRange(dataStart + i, 12).setValue(oddsByTeam[team]);
  }
}
```

Snapshot is team-keyed (not row-indexed) so post-tick re-sorts move
the odds with their team.

## HIGH #3 — `GOAL_GRID` docstring honesty (Poisson vs sim NB+DC drift)

### What was wrong

The GOAL_GRID docstring implied parity with the production simulator.
Production sim uses NB+DC marginals with team-specific dispersion
parameters; GOAL_GRID is bivariate Poisson with a single τ. On OU2.5
and BTTS at typical λ this is a 2–9pp drift — material for any
goal-market stake.

### What v2.3.3 does

Docstring rewritten to admit the deliberate drift, cite the
~9pp upper bound, and direct users to
`predictions_live.match_predictions[].{p_btts,p_over25,…}` for
high-precision goal markets. The feed exposes the sim's NB-derived
probabilities; GOAL_GRID stays as the cheap closed-form for
scenario analysis and CALIBRATE seeding.

Note: the feed does NOT expose per-team NB dispersions, so a true
NB-marginal GOAL_GRID inside Apps Script is not possible without
schema changes — out of scope for v2.3.3.

## HIGH #4 — `CALIBRATE` claimed isotonic, actually equal-width binning

### What was wrong

`CALIBRATE` docstring claimed "isotonic correction." The feed
ships pre-aggregated equal-width-bin reliability counts —
not an isotonic regressor. Independent bin-wise remaps of
(home, draw, away) routinely sum to 0.94–1.07 because the three
markets are calibrated independently, then those un-normalised
triples were written to BC:BE and read straight into EV math.

### What v2.3.3 does

1. Docstring corrected to "equal-width-bin reliability lookup."
2. Per-row renormalisation added to `_writeCalibratedProbs_`:

```js
const out = rawProbs.map(function(row) {
  const ch = _interp_(row[0], home);
  const cd = _interp_(row[1], draw);
  const ca = _interp_(row[2], away);
  const nh = Number(ch), nd = Number(cd), na = Number(ca);
  if (!isFinite(nh) || !isFinite(nd) || !isFinite(na)) return [ch, cd, ca];
  const s = nh + nd + na;
  if (!(s > 0) || !isFinite(s)) return [ch, cd, ca];
  return [nh / s, nd / s, na / s];
});
```

Falls back to raw on degenerate sums (preserves v2.3.2 behaviour on
bad input).

## HIGH #5 — `extendToKnockouts` BC:BE blank-safe formula seeds

Pre-fix, new knockout rows seeded BC:BE with bare `=I{r}/=J{r}/=K{r}`.
If λ was missing on the new row, these surfaced as `#N/A` (refreshModel
hadn't caught up yet). v2.3.3 wraps with blank-safe IF:

```js
bcOut.push([
  '=IF(I' + r + '="","",I' + r + ')',
  '=IF(J' + r + '="","",J' + r + ')',
  '=IF(K' + r + '="","",K' + r + ')',
]);
```

## HIGH #6 — `refreshAll` LockService overlap guard

Apps Script's 10-min trigger does not guarantee that a previous tick
has finished before the next starts. At UrlFetch latencies of 5–8 s
× 5 endpoints × occasional retry, ticks can overlap, interleaving
Bets/Outrights writes. v2.3.3 wraps the full tick:

```js
function refreshAll() {
  const lock = LockService.getScriptLock();
  if (!lock.tryLock(0)) {
    _writeStatus_('refreshAll: skipped — previous tick still running (10-min back-off)');
    return;
  }
  try { /* ...existing body... */ } finally { lock.releaseLock(); }
}
```

Non-blocking `tryLock(0)` so the second tick exits cleanly rather
than queuing and doubling-up later.

## HIGH #7 — `refreshIntel` no longer re-fetches `predictions_live`

`refreshAll` already fetches `predictions_live.json` once. v2.3.2's
`refreshIntel` then issued a second `_fetchJson_(predictionsUrl)` to
build the team→match map for the (broken) fan-out. Two fetches per
tick × 144 ticks/day = 144 wasted UrlFetchApp calls toward the
20,000 daily quota.

v2.3.3 threads the payload through:

```js
function refreshIntel(predictionsPayload) {
  // ...
  let predPayload = predictionsPayload;
  if (!predPayload) predPayload = _fetchJson_(_predictionsUrl_(ss));
  // ...
}
```

Fallback fetch retained for standalone invocation (operator menu,
selfTest).

## HIGH #8 — `refreshGoalGrid` clears stale cells when λ goes missing

Pre-fix, a row whose λ disappeared (e.g. mid-tournament team
withdrawal, feed regression) kept its old G:J fair-prob columns
and N:P implied/edge/BET? columns. The BET? signal would still
fire on stale fair probs vs the current bookie odds.

v2.3.3 skip path now clears:

```js
if (!sh.getRange(r, 5).getValue() || !sh.getRange(r, 6).getValue()) {
  sh.getRange(r, 7, 1, 4).clearContent();    // G:J fair probs
  sh.getRange(r, 14, 1, 3).clearContent();   // N:P implied/edge/BET?
  continue;
}
```

## MED #9 + LOW #14 — `refreshLive` PAUSES on missing timestamp / circuit breaker

Two related fail-modes that v2.3.2 didn't guard:

1. **MED #9:** `last_updated_utc` missing from the feed payload. v2.3.2
   only handled the present-but-stale case; missing was a no-op.
2. **LOW #14:** `state.warnings[]` containing a `circuit_breaker`
   string. v2.3.2 surfaced warnings cosmetically but didn't pause.

v2.3.3 collapses both into the staleKill path:

```js
const warnArr = Array.isArray(state.warnings) ? state.warnings : [];
const cbTripped = warnArr.some(function(w) {
  return typeof w === 'string' && w.toLowerCase().indexOf('circuit_breaker') !== -1;
});
if (state.last_updated_utc) {
  // ...existing stale-mins check, OR'd with cbTripped...
} else {
  staleMsg = '⚠ feed missing last_updated_utc — model paused (safe-fail)';
  if (method) {
    method.getRange(M_CELL.staleKill).setValue(true);
    method.getRange(M_CELL.autoStatus).setValue('PAUSED · feed missing timestamp');
  }
}
```

Both routes set `staleKill=true` which Method!B55 reads via OR, so
all BET? signals fail-safe to silent.

## MED #10 — `refreshModel` freshness gate accepts only finite numerics

Pre-fix gate: `if (!fresh)` — passed for `fresh = [NaN, NaN, NaN]`.
v2.3.3:

```js
if (!fresh || fresh.some(function(v) {
  return typeof v !== 'number' || !isFinite(v);
})) { ... }
```

Now sentinel strings ('n/a'), `null`, `undefined`, and `NaN` all
trigger the fall-through.

const tag = 'knockout extend: ' + conflicts + ' conflict row(s) rows ' +

## LOW #13 — `selfTest` must not write `CALIBRATION_PROP_TS`

Pre-fix, `selfTest` wrote `CALIBRATION_PROP_TS = Date.now()` before
calling `CALIBRATE(0.65, 'home')` to ensure cache warmth for the test.
Side effect: a manual selfTest run between scheduled `refreshAll`
ticks would reset the TTL clock and mask a stale feed for up to
12 h (`CALIBRATION_TTL_MS * 2 = 12 h`).

v2.3.3 validates the calibration math directly against the freshly
fetched bins (via `_interp_`) without touching ScriptProperties.
The production cache stays under `refreshAll`'s exclusive control.
