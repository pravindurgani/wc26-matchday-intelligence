## 1. Critical bug fixes (5)

### CRIT #1 — Stale-feed kill switch is now real
**v2.3 bug**: `refreshLive()` only wrote a string to `Method!B63` (cosmetic autoStatus label). `Method!B55` (the actual PAUSE gate that `Bets!AJ` reads) was never flipped. Stake formulas kept emitting on stale data.

**v2.3.1 fix**: Two changes wire the pause through end-to-end:
1. New boolean cell `Method!B81` (`STALE feed kill (auto)`, default `FALSE`).
2. `Method!B55` formula changed from
   `=AND(B53=TRUE(),$B$54<>"Y")` to
   `=OR(AND(B53=TRUE(),$B$54<>"Y"),B81=TRUE)`.
3. `refreshLive()` sets `B81=TRUE` when `live_state.last_updated_utc` is >15 min old, `FALSE` otherwise.

Result: `Bets!AJ` (engine stake) now reads PAUSE on stale feeds. Outrights `BET?` formula was extended to the same check (see HIGH #8).

### CRIT #2 — Bets BB:BI editable for shared editors
**v2.3 bug**: `applyProtections()` only listed `'AW2:AY999'` as editable on Bets. Time-trigger writes work (they fire as install owner) but manual menu-driven refreshes from a shared editor account would silently fail to write BB:BF.

**v2.3.1 fix**: Editable list now `['AW2:AY999', 'BB2:BI999']`. Covers all script-managed columns including v2.3.1's new BG/BH/BI.

### CRIT #3 — Bets BC/BD/BE no longer blank pre-install
**v2.3 bug**: BC/BD/BE were seeded as empty cells. Sheet looked "broken on first open" until first `refreshDiagnostics()` + `refreshModel()` ran.

**v2.3.1 fix**: BC/BD/BE seeded with `=I{r}`/`=J{r}`/`=K{r}` formulas at xlsx build time. On import, BC/BD/BE shows the raw model probabilities (visibly matching I/J/K). Once `refreshDiagnostics()` warms the calibration cache and `refreshModel()` runs, the formulas are overwritten with the corrected isotonic values.

### CRIT #4 — Outrights pre-populated with all 48 teams
**v2.3 bug**: Outrights shipped as a 4-row scaffold with a single placeholder row. Patch notes said "NEW Outrights sheet (48 teams × 15 cols)" — misleading.

**v2.3.1 fix**: The xlsx build script fetches `/predictions_live.json` at build time and pre-seeds the 48 team rows with current values (`p_champion`, `p_champion_p05/p95`, `p_advance`, `p_R16`, `p_QF`, `p_SF`, `p_Final`). Apps Script refreshes deltas on each run instead of creating rows from scratch.

Sheet now also includes three Group Winner / To-Qualify columns (CRIT #4 + accuracy booster — see §3 below).

### CRIT #5 — CALIBRATE quota bomb defused
**v2.3 bug**: `CALIBRATE(p, market)` used `CacheService.getDocumentCache()`. Per Google Apps Script docs, `CacheService.getDocumentCache()` returns `null` when called from custom-function context. So every recalc of `=CALIBRATE(...)` would fall through to a `UrlFetchApp.fetch()` call. With BC:BE worth of formulas (216 cells), every recalc would chew 216 of the 20,000-per-day fetch quota.

**v2.3.1 fix**:
1. `CALIBRATE()` now reads from `PropertiesService.getScriptProperties()` — which DOES work in custom-function context.
2. `CALIBRATE()` never calls `UrlFetchApp` directly. If properties are empty (first run before any refresh), the function returns the raw probability unchanged.
3. `refreshDiagnostics()` writes the calibration bins to ScriptProperties (key `wc26_calibration_v1`) with a timestamp (key `wc26_calibration_ts_v1`). 12-hour effective TTL.

Result: `=CALIBRATE(0.65,"home")` can now be safely dragged across thousands of cells; zero fetch quota cost.

## 2. High-priority fixes (3)

### HIGH #6 — Method!B10/B47/B50 row hardcodes removed
**v2.3 bug**: Method formulas hardcoded `Bets!AR2:AR73`, `Matchday!U4:U75`, `Bets!AS2:AS73` — would silently exclude knockout rounds once Bets is extended past row 73.

**v2.3.1 fix**:
- `Method!B10` (bankroll): `=B3+SUM(Bets!AR2:AR1000)`
- `Method!B47` (CLV avg): `=IF(COUNT(Matchday!U4:U1000)<5,0,AVERAGE(Matchday!U4:U1000))`
- `Method!B50` (peak): `=B3+MAX(0,IFERROR(MAX(Bets!AS2:AS1000),0))`

### HIGH #7 — Calibrated probs as decision driver (deferred, documented)
Decision math (`Bets!AH`) still reads `V/W/X` (uncalibrated blend), not `BC/BD/BE`. Switching this is a deliberate later cut once you've watched a few matchdays with the corrected values side-by-side. Don't change `AH` until you have evidence the calibrated probs are well-behaved on live results.

If you want to switch immediately: in `Bets!V2`, change `Method!$B$4*I2` to `Method!$B$4*BC2` and analogous in W/X. Then drag down. This is reversible.

### HIGH #8 — Outright BET respects PAUSE
**v2.3 bug**: Outrights `BET?` formula was just `=IF(N<Method!$B$7,"PASS","BET")`. Didn't check `Method!$B$55` PAUSE.

**v2.3.1 fix**: All 48 BET formulas now `=IF(Method!$B$55=TRUE(),"🛑 PAUSED",IF(N{r}="","",IF(N{r}<Method!$B$7,"PASS","BET")))`. Drawdown breaker AND stale-feed kill switch both pause outright signals consistently with match BET signals.

## 3. Accuracy boosters (pulled forward from v2.4)

### Bets BG/BH/BI — match context columns
Pre-populated at xlsx build time from `match_predictions[]`:
- **BG** `altitude_m` — useful for high-altitude matches (Mexico City 2,240m, Guadalajara 1,500m, Monterrey 540m)
- **BH** `climate` bucket — e.g. `high_altitude_mild`, `humid_subtropical`, `mediterranean`
- **BI** `home_travel_km` — host nation typically 0; matters more after group stage when distance compounds

These are signals the v2.3 audit flagged as silently dropped. Inspect the columns when a `WHY` text says "altitude penalty" or a match is in Mexico City.

### Outrights P/Q/R — Group Winner + To-Qualify
- **P** `p_winner_grp` — `team_predictions[].p_finish_1st_group`
- **Q** `p_2nd_grp` — `p_finish_2nd_group`
- **R** `p_3rd_place` — `p_third_place`

To bet Group Winner / Top-2 / 3rd-Place markets, look up your team's row in Outrights and compare P (or P+Q for top-2) to the bookie's implied probability. Group Winner is a tighter-margin market than 1X2 outrights at most books — the value comes faster.

Today's top group-winner picks (highest `p_winner_grp`):
- France in Group K: ~73%  ·  Spain in Group E: ~77%  ·  Argentina in Group D: ~78%
(Exact values populate on import; refresh to see current.)

### Why these and not Goals Markets (O/U, BTTS)?
Goals markets need a goal-grid engine (Dixon-Coles τ × NB-dispersion) — ~6-8h of Apps Script work and a new sheet with custom functions. Deferred to v2.4. The three boosters above were ~30 lines total in the build script and used existing schema fields.

## 7. Install instructions

### First-time install (Google Sheets)
1. **File → Import** `WC26_Value_Betting_Engine_AUTOMATED_v2.3.1.xlsx` → *Replace spreadsheet*.
2. Confirm `Live!B3/B4/B5` show the three `wc26-matchday-intelligence.vercel.app` URLs.
3. **Extensions → Apps Script** → delete any existing content → paste full `WC26_Engine_AppsScript_v2.3.1.gs` → save.
4. Reload the spreadsheet. The **WC26 Engine** menu appears.
5. **WC26 Engine → First-time setup (run once)** → approve scopes (UrlFetchApp + Properties + Triggers + Spreadsheet).
6. **WC26 Engine → Engine self-test** — expect 7 lines, all OK.

### Migrating from v2.3 (in-place upgrade)
1. **Extensions → Apps Script** → paste contents of `WC26_Engine_AppsScript_v2.3.1.gs` over your existing project → save.
2. In the spreadsheet, manually update three cells (or re-import the v2.3.1 xlsx):
   - `Method!A81`: `STALE feed kill (auto)`
   - `Method!B81`: `FALSE`
   - `Method!B55`: `=OR(AND(B53=TRUE(),$B$54<>"Y"),B81=TRUE)`
   - `Method!B10`: `=B3+SUM(Bets!AR2:AR1000)`
   - `Method!B47`: `=IF(COUNT(Matchday!U4:U1000)<5,0,AVERAGE(Matchday!U4:U1000))`
   - `Method!B50`: `=B3+MAX(0,IFERROR(MAX(Bets!AS2:AS1000),0))`
3. Run **WC26 Engine → Refresh everything** to populate the new columns.
4. Run **WC26 Engine → Lock formulas (apply protections)** to extend the protection range.

### Verifying the install
After **First-time setup**:
- `Method!B81 = FALSE` (kill flag idle)
- `Method!B55 = FALSE` (engine not paused)
- `Method!B57-B63` populated (last refresh + live state)
- `Method!B67-B79` populated (intel + diagnostics)
- `Live!B19-B26` populated (intel summary + in-play count + stale warning)
- `Bets!BB` non-zero for ~15 matches (matchday-affected today)
- `Bets!BC:BE` populated with corrected probs (slightly lower than `I:K` for home favourites in the 60-70% band)
- `Bets!BG:BI` populated with altitude/climate/travel for all 72 matches
- `Outrights` sheet: 48 teams, top of list = Spain at ~25% champion
- `Outrights` cols P/Q/R: group-winner / 2nd-place / 3rd-place probs populated
- `In-Play` sheet: 1 match right now (Brazil-Morocco)

### If stale-feed kill switch fires
- `Method!B63` shows `PAUSED · stale feed (N min)`
- `Method!B81 = TRUE`
- `Live!B26` shows `⚠ STALE feed (N min old) — model paused`
- `Bets!AJ` engine stakes drop to 0
- `Outrights!O` BET? cells show `🛑 PAUSED`

It clears automatically on the next `refreshLive()` call once the production feed catches up.

## 8. What v2.3.1 deliberately does NOT change

To minimize risk during the live tournament:
- **Decision math** — `Bets!AH/AI/AJ` formulas, `Method!B7/B8` thresholds — untouched.
- **Snapshot-on-place** — `Bets!AW:AY` + `onEdit` handler — untouched.
- **Drawdown circuit breaker** — `Method!B50-B54` — untouched (stale flag just OR's into B55).
- **De-vig switch** — `Method!B56` + `SHIN/POWER_DEVIG` — untouched.
- **Calibrated probs as decision driver** — deferred. Manual `Bets!V/W/X` switch (see HIGH #7).
