## CRIT-1 ‚Äî `installEngine()` silently succeeded on wrong workbook

### What was wrong (v2.3.6)

If the operator pasted the script into the wrong spreadsheet (a copy missing
one or more of the six required tabs ‚Äî Bets / Method / Live / Matchday /
Outrights / In-Play), `refreshAll()`'s per-step `try/catch` swallowed each
"Sheet X not found" throw, `applyProtections()` no-op'd silently on the
absent ranges, and the final "Setup complete" toast still fired. First
visible symptom: "no bets appear in column AH" 12 hours later, mid-window.

### What v2.3.7 does

`installEngine()` now calls `_preflightSheets_()` as its first action, before
any side-effect call. The helper iterates the six required `SHEET[key]`
canonical names, collects missing tabs, and on any miss:
1. Surfaces a 30-second toast titled `WC26 Engine v2.3.7 install error`
   with the missing-tab list and an operator-actionable hint.
2. Throws ‚Äî the standard Apps Script execution log records the throw so
   the install attempt is auditable.

Helper at `_preflightSheets_()` (lines 509-527).

## HIGH-1 ‚Äî Fetch errors were ephemeral (toast-only)

### What was wrong (v2.3.6)

`refreshAll()` surfaced fetch failures (`live_state`, `predictions_live`,
`matchday_intelligence`, `walk_forward`, `calibration`) via a transient
`SpreadsheetApp.toast()` only. The operator missed it if they weren't
watching the tab when the trigger ran. There was no durable cell that
said "the engine is degraded right now".

### What v2.3.7 does

Adds `LIVE_CELL.errLog = 'B28'` plus two helpers:
- `_setErrLog_(msg)` ‚Äî writes `'[UTC ISO] ' + msg` to Live!B28.
- `_clearErrLog_()` ‚Äî clears Live!B28.

`refreshAll()` calls `_setErrLog_(summary)` on any failure path, and
`_clearErrLog_()` on a fully-clean refresh. Intel, walk-forward, and
calibration fetch failures now also surface to Live!B28. Pattern mirrors
the existing `intelWarn` (B24), `staleWarn` (B26), `calibrationOob` (B27)
contract ‚Äî same Live anchor block, same render style.

## HIGH-2 ‚Äî Stale/CB banner only visible on Live tab

### What was wrong (v2.3.6)

Live!B26 (`staleWarn`) lit up when staleness ‚â• threshold or the circuit
breaker fired, but the operator's primary working surface during a window
is the **Matchday** tab. They had to flip tabs to see the warning.

### What v2.3.7 does

`refreshLive()` now mirrors the staleness/CB banner to `Matchday!W1` after
writing to Live!B26. Col W = 23 is comfortably outside the Matchday data
range (A:U = cols 1..21), so the write is collision-safe with row-1 headers
and the protected operator-input columns. Wrapped in a guarded `try/catch`
so a missing Matchday tab can't break Live refresh.

## HIGH-3 ‚Äî Quarter-Kelly stake-sizing overstated relative to walk-forward edge

### What was wrong (v2.3.6)

`Method!B5 = 0.25` (quarter Kelly) was the documented stake fraction.
The 2022 walk-forward report (`walk_forward.json` ‚Üí `lift_vs_baseline =
0.0157`, ~1.6%) shows the engine's actual edge over baseline. Kelly-optimal
for a small edge Œµ scales as Œµ/var; quarter-Kelly on a 1.6% edge is
roughly 2√ó the Kelly-optimal stake. The operator had no inline reminder
that B5=0.25 was inherited from a regime where the walk-forward edge
looked closer to ~5%.

### What v2.3.7 does

`installEngine()` calls `_seedKellyEdgeDisclaimer_()` after `applyProtections()`.
The helper reads Method!B73 (`walkForward2022Lift`, populated by
`refreshDiagnostics()` ‚Üí `refreshAll()`) and writes a human-readable
banner to `Method!C5` (immediately to the right of B5):
- If lift ‚â§ 2% ‚Üí recommend B5 ‚âà 0.10 (one-tenth Kelly), naming the lift.
- Otherwise ‚Üí confirm B5 = 0.25 quarter-Kelly is consistent with the
  observed edge.

Non-destructive: B5 itself is left alone ‚Äî the operator decides whether
to adjust.

## MED-1 ‚Äî No per-endpoint freshness audit trail

### What was wrong (v2.3.6)

Method had a single `lastDiagnosticsRefresh` cell. There was no way for
the operator (or a post-mortem) to see "the intel endpoint last responded
200-OK at 14:32:07Z, but the live endpoint hasn't responded since 13:08Z" ‚Äî
the kind of diagnostic that distinguishes a slow upstream from a hung
trigger.

### What v2.3.7 does

Five new Method anchors (all in the previously-unused B82..B86 band ‚Äî
B81 is `staleKill`):

```
Method!B82 ‚Üí endpointLiveTs       (live_state.json)
Method!B83 ‚Üí endpointPredsTs      (predictions_live.json)
Method!B84 ‚Üí endpointIntelTs      (matchday_intelligence.json)
Method!B85 ‚Üí endpointCalibTs      (calibration.json)
Method!B86 ‚Üí endpointWalkFwdTs    (walk_forward.json)
```

`_stampEndpoint_(cell)` writes `new Date()` and is called immediately
after each successful `_fetchJson_()`. On fetch failure the corresponding
cell is intentionally left untouched ‚Äî its age IS the diagnostic.
