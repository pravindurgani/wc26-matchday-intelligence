1B
Net delta injury calculation in injury_adjustments.py
File: scripts/live/injury_adjustments.py, scripts/live/apply_matchday_adjustments.py
Do second

Change the calculation from "full penalty for the player who's out" to "penalty = player_out.elo − replacement.elo". The existing discounted_elo() function returns the out-player's contribution. Add a second lookup for the replacement.

def net_injury_elo(
    tier_out: str,
    status: str,
    replacement: dict | None,
) -> float:
    """
    Net Elo impact = out-player cost + replacement quality.
    
    Example: Mbappe (tier_1_star=-30) replaced by Dembele (tier_2_starter=-12)
    → net = discounted(-30) + replacement_elo(-12) = -30 + (-(-12)) = -18
    
    If no replacement data: conservative assumption = 0 (neutral replacement).
    If replacement is unknown youth: use squad_value ratio * tier penalty.
    """
    out_elo = discounted_elo(tier_out, status)     # existing function, unchanged
    
    if replacement is None:
        replacement_elo = 0.0                        # conservative: neutral replacement
    else:
        replacement_elo = replacement.get("elo_equiv", -5.0)
    
    # Net = what we lose from star - what we gain from replacement
    # Both are negative; we SUBTRACT replacement (reducing the penalty)
    net = out_elo - replacement_elo
    return net   # still negative, but smaller than raw out_elo

Where to integrate

In apply_matchday_adjustments.py → _load_injury_components(), after looking up the player's tier, call net_injury_elo(tier, status, player.get("replacement")) instead of discounted_elo(tier, status). The replacement field comes from the expanded key_players_2026.json lookup in classify_tier().

Phase 2 — Referee Intelligence

Free signal. FIFA publishes referee assignments per match. The API already returns it. It's just never consumed.
Why this matters for knockouts especially: In group stage matches between mismatched teams, referee variance is low — the stronger team wins regardless. In knockout matches (QF, SF, F), the gap between teams is smaller and referee tendencies have a measurable effect on outcomes. A referee who gives 30% more penalties than average in close games is a real edge.

  • pen_rate_per_match: directly affects xG; a high-pen ref in a match between an aggressive pressing team and a physical defender is a material signal

Minimum sample: require n_matches ≥ 20 before including a referee entry. Below 20 matches, default to 0 adjustment.

Add a true_xg_available field to each match record in match_stats_2026.json — currently it's hardcoded to false by spec. With this change, it becomes an actual runtime check. The validator and dashboard can surface this.

The xG-to-Elo scale factor

A 1.0 xG delta (team dominated by 1.0 expected goals) should map to approximately 8–10 Elo points. The model's λ predictions show that a 10-Elo difference shifts the expected goal difference by ~0.12. So 1.0 xG ÷ 0.12 = ~8.3 Elo per xG unit. Use 8.0 as the scalar, still clamped at ±STATS_PROXY_RAW_CAP (12.0).

4B
Position-aware lineup adjustment
File: scripts/live/lineup_adjustments.py
Medium value

Current heuristic: 3+ outfield changes = −3 Elo (flat, position-blind). Upgrade to position-weighted: losing attacking players matters more than losing defensive ones because attacking quality drives expected goals more directly.

# Updated compute_lineup_delta_elo() — add position weighting
POSITION_WEIGHTS = {
    "F": 1.0,   # Forward rotation: full weight
    "M": 0.7,   # Midfielder rotation: 70% weight
    "D": 0.4,   # Defender rotation: 40% weight
    "G": 0.0,   # GK handled by existing GK_SWAP_ELO separately
}

def compute_weighted_rotation_penalty(prior_xi, current_xi) -> float:
    outfield_changes = _outfield_diff(prior_xi, current_xi)
    total_weight = sum(POSITION_WEIGHTS.get(p.get("pos"), 0.7)
                      for p in outfield_changes)
    if total_weight < 2.0: return 0.0           # light rotation: no penalty
    return max(-8.0, -total_weight * 0.8)        # cap at -8 (existing LINEUP_CAP allows to -20)

This means: 3 defenders rotated out = ~−0.96 Elo (small). 3 attackers rotated out = ~−2.4 Elo. GK swap + 3 attackers = −8 + −2.4 = −10.4 Elo (major rotation detected).

New sheet: GoalGrid in spreadsheet; new function GOAL_GRID() in Apps Script
High commercial value

Building the Goal Grid is mechanical: use the NB-dispersion parameters (already in the model) to compute the full score matrix and derive market probabilities.

The GOAL_GRID custom function

/**
 * GOAL_GRID(lam_h, lam_a, market) — compute fair probability for goals markets
 * @param lam_h  Expected goals for home team (from predictions_live.json λ fields)
 * @param lam_a  Expected goals for away team
 * @param market "ou25" | "btts" | "ou15" | "ou35" | "ah0" | "cs{X}{Y}"
 * @returns Fair probability (0-1) before de-vig
 */
function GOAL_GRID(lam_h, lam_a, market) {
  // Build score matrix P(h_goals, a_goals) using Poisson + Dixon-Coles τ=0.10
  const MAX_GOALS = 8;
  const TAU = 0.10;     // from 03_simulate.py dc_correction()
  const matrix = buildScoreMatrix(lam_h, lam_a, MAX_GOALS, TAU);
  
  if (market === "ou25") return sumMatrix(matrix, (h,a) => h+a > 2.5);  // Over 2.5
  if (market === "btts") return sumMatrix(matrix, (h,a) => h > 0 && a > 0);
  if (market === "ou15") return sumMatrix(matrix, (h,a) => h+a > 1.5);
  if (market === "ou35") return sumMatrix(matrix, (h,a) => h+a > 3.5);
  // AH -0.5 (home wins outright)
  if (market === "ah0")  return sumMatrix(matrix, (h,a) => h > a);
  return 0;
}
